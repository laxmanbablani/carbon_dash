## New Tasks: Package Rename to `carbon_dash_components`

### Rationale
User requests the project name to be `carbon_dash_components` (Python module name). Distribution name on PyPI should be `carbon-dash-components` (hyphens). The current `carbon_dash` will be replaced.

### Rename Steps (atomic; all Must be done together)

1. **Rename directory**:
   - `carbon_dash/` → `carbon_dash_components/`

2. **Update `carbon_dash_components/package-info.json`**:
   - Change `"name": "carbon_dash"` → `"name": "carbon-dash-components"`

3. **Update `setup.py`** (at repository root):
   - Use raw package.json name for PyPI distribution
   - Derive Python module name for `packages` list
   ```python
   import json
   from setuptools import setup
   from pathlib import Path

   here = Path(__file__).parent
   with open('package.json') as f:
       package = json.load(f)
   long_description = (here / 'README.md').read_text()

   dist_name = package["name"]  # raw name for PyPI
   module_name = package["name"].replace(" ", "_").replace("-", "_")

   setup(
       name=dist_name,
       version=package["version"],
       author=package['author'],
       packages=[module_name],
       include_package_data=True,
       license=package['license'],
       description=package.get('description', dist_name),
       long_description=long_description,
       long_description_content_type="text/markdown",
       install_requires=['dash>=2.0.0'],
       classifiers=['Framework :: Dash'],
   )
   ```

4. **Update all test imports** in `tests/*.py`:
   - `import carbon_dash as cd` → `import carbon_dash_components as cd`

5. **Update `package.json` scripts**:
   - `"build:backends": "dash-generate-components ./src/lib/components carbon_dash_components -p package-info.json --ignore '\\.test\\.'"`
   - `"build:wheel": "python setup.py sdist bdist_wheel"` (unchanged)

6. **Update CI and publish workflows**:
   - In `.github/workflows/ci.yml` and `.github/workflows/publish.yml`, the command `dash-generate-components ./src/lib/components carbon_dash ...` should target `carbon_dash_components`
   - Also ensure any other references to `carbon_dash` as import name are updated (none expected outside tests)

7. **Move `package-info.json`**:
   - It lives inside the Python package directory; renaming the folder automatically handles this.

8. **Regenerate backends**:
   - Run `npm run build:backends` after renaming to generate Python files into `carbon_dash_components/`

9. **Run tests locally** to confirm everything imports correctly before pushing.

### Important Notes
- The internal `__init__.py` uses `__name__` to derive paths; it will work after rename.
- JS namespace derived from `package_name` (the underscore version) will become `carbon_dash_components` — acceptable.
- The `build:js` webpack output remains in `build/` unchanged.

## Add pyproject.toml

Create `pyproject.toml` at repo root:

```toml
[build-system]
requires = ["setuptools>=61.0", "wheel", "build"]
build-backend = "setuptools.build_meta"

[project]
name = "carbon-dash-components"
version = "0.0.1"
description = "Carbon React Components wrapped for Dash"
readme = "README.md"
authors = [{name = "Laxman Bablani", email = "laxman@bablani.org"}]
license = {text = "MIT"}
requires-python = ">=3.8"
classifiers = ["Framework :: Dash"]
dependencies = ["dash>=2.0.0"]

[project.urls]
Homepage = "https://github.com/laxmanbablani/carbon_dash"
Repository = "https://github.com/laxmanbablani/carbon_dash"

[tool.setuptools]
packages = ["carbon_dash_components"]
include-package-data = true
```

- The `name` field matches the desired PyPI distribution name (`carbon-dash-components`)
- The module name (`carbon_dash_components`) is specified under `[tool.setuptools] packages`
- `setup.py` remains for `dash-generate-components`; `pyproject.toml` does not replace it
